# Bug Hunt Findings — observablecafe

Generated by 5 subagents reviewing the workspace, then independently verified by 3 red-team reviewers.
Each finding carries a red-team verdict: CONFIRMED / WEAK / FALSE POSITIVE.

## ⚠️ Red-team corrections to the original report

- **FINDING #1 (context.rs "assitan" typo) — FALSE POSITIVE.** Line 28 actually reads
  `Some("user") | Some("assistant")` — correctly spelled. The misspelling existed only in
  the original report, not the source. DROPPED.
- **FINDING #11 (publish-before-subscribe in executor.rs/tool_executor.rs/chat.rs) — FALSE POSITIVE.**
  The chunk published *before* subscribe is the request; the awaited *response* is published after
  subscribe and is retained 60s (`with_retain(60)`), so it is replayed via `drain_retained`. DROPPED.

Verdict legend: ✅ CONFIRMED · 🟡 WEAK (real but low impact / mislabeled) · ❌ FALSE POSITIVE.

## Critical (feature-breaking, silent)

2. ✅ **`cafe-http-proxy-sdk/src/lib.rs:65-71`** — `parse_registration` calls `.as_str()` on a JSON *object* annotation → always returns `None`. Entire HTTP-proxy route registration (`cafe-web-fetch`) silently disabled. (Red team confirmed `with_annotation` stores a JSON object; `cafe-server/src/main.rs:139` never registers.)
3. ✅ **`cafe-cli/src/main.rs`** (16-17, 602-604, 625-627) — read-credential lookups use `"binary.read_url"`/`"binary.read_token"` instead of `"cafe.binary.read_url"`/`"cafe.binary.read_token"` (the actual key emitted at `cafe-binary-store/src/main.rs:186`). `store download`/`stream` always fail "no read credentials found".
4. ✅ **`cafe-knowledgebase/src/index.rs:362-376`** — `chunk_text` infinite-loops for any long doc (default 512/64 config). `start=end` never breaks → hangs indexer. Red-team traced `len=100,cs=60,ov=20` confirming the loop never terminates.

## High (wrong behavior / crashes)

5. ✅ **`cafe-knowledgebase/src/index.rs:287-294`** — `delete` only removes parent row; `…--chunk-N` rows orphaned (stored with `chunk_id = "{doc_id}--chunk-{i}"`, `parent_doc_id = doc_id`), still returned by search.
6. ✅ **`cafe-dice/src/main.rs:82-88`** — `dice.roll` panics on `sides=0` (empty range `1..=0`); `count` unbounded. Direct RPC bypasses the detector's `sides>=1` guard. (Red team: panic CONFIRMED; unbounded count WEAK — bus-internal, not internet-facing.)
7. ✅ **`cafe-server/src/db.rs:128`** — `ensure_admin_token` uses `fetch_one`; if tokens exist but none admin, errors and aborts `main()`.
8. 🟡 **`cafe-mcp-client/src/main.rs:152`** — `call_id_counter` reset to `3` inside the lock every call → every tool call uses id `4`; a stale id-4 response after a timeout is consumed as the next call's result. (Red team: real but narrow — serialized under lock in normal flow.)
9. 🟡 **`cafe-binary-store/src/main.rs:254`** — `offset + data.len() as u64 - 1` underflows (panic in debug) on empty/EOF reads. Only consumed in `offset>0` branch, so harmless in release builds.
10. ✅ **`cafe-binary-store/src/main.rs:197`** — multipart upload finalized after first part (`offset==0 && !empty`); second parts 404. No working multi-part path.

## Medium (races / leaks / correctness)

16. ✅ **`cafe-stt/src/main.rs:322-331`** — `convert_to_wav` leaks ffmpeg on timeout (no `child.kill()`).
17. 🟡 **`cafe-sdk/src/http/sse.rs:14-19`** — SSE parser drains buffer on JSON parse failure (lowers severity: only the malformed line is dropped, not all subsequent bytes).
18. ✅ **`llm-cafe/llm_cafe/client.py:98,128`** — strict `decode("utf-8")` of network chunks → `UnicodeDecodeError` on split multibyte UTF-8.

# Why tests didn't catch these bugs

Compiled from 3 coverage-gap reviewers who read the actual tests. Each bug maps to a root cause.

## Per-bug: why the test gap exists

**`parse_registration` → proxy routes never register (http-proxy-sdk)**
- `lib.rs` has NO `#[cfg(test)]`; `cafe-server/src/route_registry.rs` tests call `upsert` directly, skipping the broken parser. No E2E publishes a `RouteRegistration` and asserts a proxied request succeeds.
- Gap: untested function + tests call the downstream step directly, bypassing the defect.

**`cafe-cli` read-cred key mismatch**
- `cafe-cli` has ZERO tests. `tests/binary-store-e2e.py` speaks HTTP/JWT directly via `urllib` and never runs `cafe-cli store download/stream`. Production consumers (`cafe-stt`, `cafe-tts`) use the correct key, so the CLI path is never exercised in real flows.
- Gap: untested crate + E2E bypasses the buggy code entirely.

**`chunk_text` infinite loop (knowledgebase)**
- `index.rs` unit tests exist but `chunk_text_hard_split_long` uses `overlap=0`. The infinite loop requires `overlap>0` AND a long single paragraph (the crate's actual default 512/64). No test combines them.
- Gap: happy-path fixtures only; the exact triggering input combo is never generated.

**`delete` orphans chunk rows (knowledgebase)**
- No `delete`/index-integration test at all. Nothing indexes a doc, calls `delete`, then searches and asserts zero results.
- Gap: entire index→delete→search lifecycle untested.

**`dice.roll` panic on `sides=0` / unbounded `count`**
- Tests cover only `parse_roll` (detector). `dice.roll` RPC handler is never called directly with `sides=0` or huge `count`. `tool-calling-e2e.py` only sends `!roll 2d6`. The detector guard can't be triggered, and tests never reach the unguarded handler.
- Gap: only the parser tested, not the executor; no negative input at the RPC layer.

**multipart upload finalized after first part (binary-store)**
- `storage.rs` unit tests don't model the handler's finalize decision. `binary-store-e2e.py` only does single-shot POST (whole body, `offset=0`), where finalizing is correct.
- Gap: E2E exercises only the single-shot path; multi-part fragmentation never triggered.

**u64 underflow on empty/EOF read (binary-store)**
- `Storage::read` tests use non-empty data; E2E reads `b"Hello!"`. No test writes an empty file or triggers a 0-byte read.
- Gap: no empty/edge-input test.

**`ensure_admin_token` aborts main() (server db)**
- `cafe-server/src/db.rs` has NO test module. No E2E seeds tokens-without-admin and restarts.
- Gap: untested function; abort path never exercised.

**`upsert_session_with_tags` omits agent_id (store db)**
- `cafe-store/src/db.rs` has NO test. `ephemeral-sessions-e2e.py` sets `agent_id` once and tests lifecycle, never re-upserts with a different `agent_id`.
- Gap: no test asserts the conflict-update actually mutates `agent_id`.

**abort non-functional (evaluator.rs)**
- No unit test references abort. `SessionEvaluator` abort plumbing is `#[allow(dead_code)]` and never wired into `run_session`. No E2E publishes an abort `Null` chunk mid-stream.
- Gap: untested feature + `dead_code` attribute hid the disconnected channels.

**JSON-negotiated `SetMeta` role dropped (bus client)**
- `ephemeral-sessions-e2e.py` `BusConnection` sends role in a SEPARATE second `set_meta` (via `client_loop`), re-populating the role and masking the dropped JSON-negotiation branch. No test sends `codecs` + role in the initial handshake and asserts the role stuck.
- Gap: two-step re-send masks single-shot init defect; no assertion on post-handshake `conn_meta`.

**`url_decode` UTF-8 corruption (proxy.rs / mcp-bridge http.rs)**
- ZERO unit tests for `url_decode` (neither copy). Not referenced by any test. Only exercised incidentally with ASCII params.
- Gap: pure function never unit-tested, incl. `%E2%82%AC` / malformed `%`.

**SSE sessions leak 3600s (mcp-bridge)**
- `SseSessions` is a private in-process map; no log/metric/API exposes insert/remove, so black-box E2E can't observe it. `mcp-bridge-e2e.py` checks tool results, not session lifecycle.
- Gap: unobservable internal state; cleanup logic completely untested.

**`call_id_counter` reset → stale response (mcp-client)**
- `cafe-mcp-client` has no `#[cfg(test)]`. `mcp-client-e2e.py` uses a fake server that always responds immediately/successfully — never times out, never issues a SECOND call.
- Gap: no timeout + no second-call test; id reuse/stale-response hazard unobserved.

**`isError`/non-JSON content mishandled (mcp-client)**
- `mcp-client-e2e.py` only happy-path (`result.content[0].text` = valid JSON). Note `mcp-bridge-e2e.py:157` asserts `isError` — but on the *bridge*, not the *client* binary. Cross-boundary attribution gap.
- Gap: no test sends `isError:true` / `error` object / non-text content to the client.

**`convert_to_wav` leaks ffmpeg on timeout (stt)**
- `cafe-stt` has NO `#[cfg(test)]`. `stt-e2e.py` feeds only valid WAV; and on voicebox error it prints a "partial" non-failing message (see below), so even conversion errors don't hard-fail.
- Gap: timeout branch never hit + graceful partial masks failures.

**SSE parser drains on JSON failure (sdk sse.rs)**
- `sse.rs` has tests but none feed `data: <bad json>\n<good json>\n` and assert the good one survives. `parse_never_panics` feeds random bytes but not that sequence.
- Gap: missing negative/edge test for the failure-drain branch.

**strict UTF-8 decode split chunk (llm-cafe client.py)**
- `llm-cafe` has NO test files. No test injects a multibyte char split across HTTP chunk boundaries.
- Gap: untested package; streaming-boundary assumption never stressed.

**`QuickiesPanel` doubled/never-rendered streaming text (web)**
- `cafe-web` has NO test suite (`package.json` has no test runner; no `*.test.tsx`). Source proves the duplication (per-token deltas + full `response_chunk` both counted; `+ chunk.content` double-counts).
- Gap: no frontend tests at all.

**`stt-e2e.py` false pass on voicebox outage (test itself)**
- This IS the test, and it's NOT in CI (`ci.yml` runs only bus-filters, source-connection, ephemeral-sessions, tts). On error it prints "partial" and exits 0 with no `assert`/`sys.exit(1)`. CI never greps for `=== ALL ... PASSED ===`.
- Gap: flawed self-check (zero-tolerance violation) + not wired into CI + no pass-string guard.

**`bufio.Scanner` 64KB aborts SSE (telegram client.go)**
- NO Go test files exist anywhere (`cafe-telegram` is a separate Go module, excluded from `cargo test --workspace`). No test feeds a >64KB SSE line.
- Gap: entire Go bridge untested + outside CI.

**nil `msg.From` panic (telegram bot.go)**
- No Go tests. `b.api` is a concrete `tgbotapi.BotAPI` with no mock seam, so `handleUpdate`/`isTrusted` can't be unit-tested. `Run` guards `Message==nil` but not `From==nil`.
- Gap: untested + missing mock interface.

**`Send` errors ignored / no 4096 split (telegram bot.go)**
- No Go tests. No mock of `b.api.Send` returning an error; no assertion that long replies split.
- Gap: untested + missing mock seam.

**`Scan` errors skipped (telegram db.go)**
- No Go tests. No injection of a row that fails `Scan` and asserts error.
- Gap: untested.

**`id, _ := result["id"].(string)` returns "" (telegram client.go)**
- No Go tests. No test feeds a response missing `"id"` and asserts an error.
- Gap: untested.

**`bus-filters-e2e.py:99` accepts `content_type == None` (test itself)**
- It IS in CI, but the assertion `ct in ("text", None)` explicitly permits `None`. No negative test asserts the filter *rejects* null-type chunks.
- Gap: over-permissive assertion + no negative test.

## Cross-cutting themes

1. **Entire crates/areas have zero tests.** `cafe-cli`, `cafe-server/src/db.rs`, `cafe-store/src/db.rs`, `cafe-stt`, `llm-cafe`, the whole `cafe-telegram` Go bridge, and the entire `cafe-web` frontend have no automated tests. The bugs live in untested code.
2. **Tests call the safe helper, not the broken step.** `parse_registration`/`route_registry`, `parse_roll`/`dice.roll` — tests exercise the well-behaved upstream function directly, skipping the defective downstream path. Green tests, broken runtime.
3. **Happy-path / single-shot fixtures only.** `chunk_text` (overlap=0), multipart (single-shot), binary reads (non-empty), dice (`2d6`) — the exact input combo that triggers each bug is never generated.
4. **No negative / error-injection / adversarial-boundary tests.** No `sides=0`, no empty read, no `isError`, no `Send` error, no null content_type, no multibyte-split, no >64KB line, no timeout.
5. **Two-step re-send masks init defects.** The bus e2e re-sends `set_meta` with the role, hiding the dropped JSON-negotiation branch; the SSE parser's bad-line drain is benign for the trivial two-line case.
6. **Graceful degradation defeats "zero silent failures."** `stt-e2e.py` prints a "partial" and exits 0 — a direct violation of the repo's own E2E rule. No CI guard on the exact `=== ALL ... PASSED ===` string.
7. **Unobservable internal state.** SSE session-map leak and ffmpeg-process leak live in private structures / OS tables with no external signal a test could assert on.
8. **Missing mock seams.** `tgbotapi.BotAPI` is concrete (no interface), making `handleUpdate`/`isTrusted`/`streamToTelegram` structurally hard to unit-test.
9. **`#[allow(dead_code)]` hid a disconnected feature.** The evaluator's abort plumbing was flagged dead and never wired, so the abort bug was invisible to both tests and reviewers.
10. **Cross-boundary coverage illusion.** `isError` is tested on `cafe-mcp-bridge` but the bug is in `cafe-mcp-client`; grep-based "coverage" looks present but the defective path is untested.
